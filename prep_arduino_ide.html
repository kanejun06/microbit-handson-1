WHYなぜこの開発環境が必要なの？

今回のハンズオンの全体像と、Arduino IDE の役割

今回つくるのは、「カメラの前にあるものを、機械が自分で見分ける」仕組みです。たとえばカメラに映ったものが「リンゴ」なのか「バナナ」なのかを判定し、その結果を micro:bit に送ってディスプレイに表示します。

この「見分ける頭脳」を動かすのが ESP32S3 という小さなカメラ付きマイコンです。ただ、ESP32S3 は micro:bit のようにブラウザだけでは書き込めません。プログラムを書いて、機械が読める形に翻訳（コンパイル）し、基板に流し込む——この一連の作業をするための道具が必要です。それが Arduino IDE です。

つまり Arduino IDE は、この鎖の一番左にある「土台」です。ここが用意できていないと、当日いくら良いプログラムやモデルがあっても ESP32S3 に送り込めません。だから事前に入れておいてほしい、というわけです。

💡 micro:bit と何が違うの？
micro:bit は makecode.microbit.org というウェブサイト上だけでプログラムが完結しました。一方 ESP32S3 は、より本格的な処理（カメラ画像の機械学習など）をするぶん、PCにインストールする専用ソフト（＝Arduino IDE）が必要になります。「ブラウザでお手軽」から一歩進んだ、本格的な開発環境を体験する回でもあります。

LEARNその前に：機械学習ってなに？

インストールの前に、これから自分が何を作るのかをイメージしておこう

「機械が自分で見分ける」と言いましたが、その裏にあるのが 機械学習（Machine Learning） という考え方です。

CONCEPT機械学習＝「ルールを教える」のではなく「例から学ばせる」

ふつうのプログラムは、人間が「もし〜ならこうしろ」というルールを全部書いて動かします。でも「リンゴとバナナを見分けるルール」を言葉で完璧に書くのは、ほぼ不可能ですよね（色？ 形？ 例外だらけです）。

そこで機械学習では、ルールを書く代わりに 「これがリンゴ」「これがバナナ」というお手本（データ）をたくさん見せて、機械自身に共通点を見つけさせます。 この「見せて覚えさせる」作業を 学習（トレーニング） と呼びます。学習の結果できあがった「見分ける能力のかたまり」が モデル です。

今回は、写真をたくさん見せて学習させる作業を Edge Impulse というウェブサービスで行います。難しい数式を書く必要はなく、画像を集めてボタンを押していくだけでモデルが作れます。

画像の機械学習には、大きく2つのタイプがある

今回扱う Classification（分類） と、よく対比される Object Detection（物体検出） の違いを知っておくと、自分が何を作っているのか迷いません。

🏷️ 分類
Classification

画像ぜんぶを見て「これは何か」を1つ答える。「写っているのはリンゴ？ バナナ？」のように、画像全体に1枚のラベルを貼るイメージ。

「どこにあるか」は分からない。「何が写っているか」だけを答えます。

例：カメラの前のものが「リンゴ」か「バナナ」か「何もなし」かを判定
🎯 物体検出
Object Detection

画像の中の「どこに・何が」あるかを、枠で囲って答える。1枚の画像に複数のものが写っていても、それぞれの位置を四角い枠（バウンディングボックス）で示します。

位置まで分かるぶん、学習も処理も重くなりがち。

例：棚の写真から「ここにリンゴ、あそこにバナナ」と位置つきで検出

👉 今回のハンズオンで使うのは Classification（分類）。「カメラの前に何があるか」を1つ答えるシンプルなタイプです。

で、Arduino から ESP32S3 に「何を」書き込むの？

ここが今回の肝です。ESP32S3 に流し込むのは、ただのプログラムだけではありません。学習済みのモデルごと書き込みます。

📦 ESP32S3 に書き込まれる中身

  • ① カメラを動かすプログラム：カメラで写真を撮り、決まったサイズに整える部分
  • ② 学習済みモデル本体：Edge Impulse で作った「見分ける能力」。これがライブラリという形でプログラムに組み込まれます
  • ③ 判定して結果を送る処理：撮った画像をモデルに通し、「一番それらしいラベル」を選んで micro:bit へ送る部分

つまり、ネットにつながっていなくても ESP32S3 単体で判定できます。クラウドに問い合わせるのではなく、機械学習の「頭脳」そのものを基板の中に焼き込む——これがエッジAI（端末側で完結するAI）の面白さです。

💡 ここまでのまとめ
Edge Impulse でモデルを作る（学習）→ それをライブラリとして書き出す → Arduino IDE でカメラ用プログラムと一緒に ESP32S3 へ書き込む。今日入れてもらう Arduino IDE は、この最後の「書き込む」を担当する道具です。全体像が見えたところで、さっそく準備しましょう。

💡 ダウンロードページでの注意
寄付（Donate）を求める画面が出ますが、「JUST DOWNLOAD」を選べば無料でダウンロードできます。「Windows Win 10 and newer, 64 bits」のように、自分のOSに合った行を選んでください。

② ESP32 ボードを追加する

Arduino IDE はインストール直後の状態では ESP32S3 を扱えません。下の手順で「ボード定義」を追加します。

  1. Arduino IDE を起動し、設定（Preferences）を開く
    Windows：File > Preferences ／ macOS：Arduino IDE > Settings…
  2. 「追加のボードマネージャのURL」に下記を貼り付けて OK
    Additional Boards Manager URLs という欄です。右側の小さなアイコンを押すと入力窓が開きます。
    https://espressif.github.io/arduino-esp32/package_esp32_index.json
  3. ボードマネージャを開く
    左端のボードのアイコン、または Tools > Board > Boards Manager
  4. 検索欄に esp32 と入力し、「esp32 by Espressif Systems」を見つける
    ※ 似た名前の「Arduino ESP32 Boards」ではなく、Espressif Systems の方を選んでください。
  5. バージョン 2.0.4 を選んで Install
    バージョンを選ぶプルダウンがあります。今回のサンプルは 2.0.4 で動作確認済みのため、最新版ではなく 2.0.4 を選ぶのが安全です。ダウンロードに数分かかることがあります。
  6. インストールが終わったら準備完了！
    ボードの選択（ESP32S3の指定）は当日いっしょにやります。ここまでできていればOKです。
